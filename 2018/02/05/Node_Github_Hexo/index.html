至此本地浏览器输入网址即可访问

把本地的hexo提交到giehub

  1. 注册gitHub账号（注册地址），以为下面要用到用户名及邮箱，所以假设用户名是：bamboocup 邮箱是：bamboocup@163.com去注册

  2. 登录创建并创建仓库（创建地址）,仓库名字务必是xxx.github.io，xxx是指你想起的名字，创建后浏览器输入bamboocup.github.io即可访问你的仓库内容

  3. 配置git（下面的Git bash黑窗下操作）

    使用git上传内容到GitHub，首先要设置Git的用户名和邮件地址（邮箱就是你注册Github时候的邮箱），让GitHub知道上传东西时候来者何人

    $ git config --global user.name "bamboocup"
$ git config --global user.email "bamboocup@163.com"

  4. 本地git与GitHub建立联系，git与GitHub是通过ssl提交代码，所以判断电脑是否已经生成ssh（下面的Git bash黑窗下操作）

$ ls -al ~/.ssh                                检查电脑是否已经有SSH
$ ssh-keygen -t rsa -C "bamboocup@163.com"    一直回车即可生成.ssh目录，里面有id_rsa和id_rsa.pub两个文件，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。    
$ clip < ~/.ssh/id_rsa.pub                    此命令是复制公钥内容（待会粘贴用）

  1. 本地git与GitHub建立联系，打开GitHub网址在github右上角上点击你的头像–>Settings–>SSH and GPG keys–>New SSH key（下面的Git bash黑窗下操作）

    $ ssh -T git@github.com 测试是否已经配置成功

    Hi bamboocup! You've successfully authenticated, but GitHub does not provide shell access.

![](https://i.imgur.com/hxtO5xB.png)
![](https://i.imgur.com/9l9MdMq.png)

  1. 下面修改hexo的配置把博客的文件提交到GitHub上，在E盘的hexoBlog文件夹下打开_config.yml文件，修改deploy的值，repository的值在GitHub新增的bamboocup.github.io仓库的页面可以获得，可以使用ssh的值也可以使用https的值

    deploy:
  type: git
  repository: git@github.com:bamboocup/bamboocup.github.io.git
  branch: master

  1. 把文件提交到GitHub上（下面的命令cmd.exe黑窗下操作）

    E:\hexoBlog> npm install hexo-deployer-git –save 在生成以及部署文章之前，需要安装一个扩展, 没安装不能执行后面的hexo deploy命令

    npm WARN deprecated swig@1.4.2: This package is no longer maintained

    E:\hexoBlog> hexo generate 生成静态文件（hexo g简写命令也是同样的）

    E:\hexoBlog> hexo deploy 因为上面配置了deploy的repository此命令会把代码提交到GitHub

  2. 至此整个创建过程完毕，有不足之处请见谅！
