windows下安装PHPStudy2018和memcached服务已经添加PHP7.2.10 NTS对应的memcache.dll和Redis.dll扩展

php memcached 2018/02/09

1、下载安装PHPStudy 2018(下载地址)，下载完安装到D:/phpstudy

2、下载和安装Windows下的memcached服务与redis服务

下载memcached.exe

32bit：memcached-win32-1.4.4-14.zip（单击直接下载）

64bit：memcached-win64-1.4.4-14.zip（单击直接下载）

上面的下载地址来源亚马逊的XMLs3.amazonaws.com/downloads.northscale.com/

把下载好的压缩包解压到D:/memcached目录下然后安装

Microsoft Windows [版本 6.1.7601]

C:\Windows\system32>d:
D:\>cd memcached
D:\memcached>memcached.exe -d install
D:\memcached>memcached.exe -d start
D:\memcached>netstat -aon | find "11211"
  TCP    0.0.0.0:11211          0.0.0.0:0              LISTENING       6340
  TCP    [::]:11211             [::]:0                 LISTENING       6340
  UDP    0.0.0.0:11211          *:*                                    6340
  UDP    [::]:11211             *:*                                    6340
D:\memcached>


下载并安装redis.msi

目前提供下载的安装包是64位系统的，github.com/MicrosoftArchive/redis,下载完一直下一步知道安装完成即可。


以及下载redis软件服务

添加一个虚拟空间然后里面放一个内容有<?php phpinfo()?>的index.php，记得注意里面编译器的版本是x86,已经PHP的扩展是NTS和VC15因为这个决定后面要下载的扩展的版本



下载memchche扩展和redis扩展

本来想到是去php官方的扩展库下载(pecl.php.net/package-stats.php和windows.php.net/downloads/pecl/releases/)，但是里面memcache扩展的版本太低最后搜受到GitHub上面有php7的扩展（github.com/nono303/PHP7-memcache-dll），下载时务必注意你php时x86且是7.2且是NTS（非线程）,下载完后把memcache.dll和redis.dll文件放到D:/phpstudy/PHPTutorial/php/php-7.2.1-nts/ext下（刚刚phpstudy安装决定存放位置）,然后打开D:/phpstudy/PHPTutorial/php/php-7.2.1-nts的php.ini把扩展引入加上extension=php-7.2.x_memcache.dll extension=php_redis.dll，最后重启服务。






ps：在windows下安装的memcached.exe和redis.msi是php能操作计算机内存的桥梁，相当于我们平时说驱动的意思，它们安装后是windows的服务，而php有memcache、memcached、redis扩展，这里重点说一下memcache和memcached扩展，它们之前有点名字上只差了个d，它们提供的操作也类似，但是memcached更加丰富的函数，其中cas是memcached有memcache没有的，memcached扩展比memcache扩展一个非常重要的优势， 在这样一个系统级（memcache自身提供）的冲突检测机制（乐观锁）下， 我们才能保证高并发下的数据安全，memcache和memcached都是依赖memcached服务而操作内存，而在window系统下暂时没找到memcached的扩展，仅能在linux下运行
