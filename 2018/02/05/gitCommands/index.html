git常用命令

2018/02/05

git常用命令

创建仓库

$ mkdir learnGit  创建目录
$ cd learnGit      进入目录
$ pwd              显示当前目录路径
$ git init          初始化仓库
$ ls -ah          查看.git隐藏文件生成情况

工作区添加文件并提交到版本库

$ pwd                                    显示当前目录路径必须是创建版本库的同目录
$ git status                            版本库的当前变化状态（无）
$ echo 'version1 text content' > v1.md  文件里面写入内容（文件没则新建有则覆盖）
$ ll                                    查看文件生成情况
$ cat v1.md                                查看文件内容
$ git status                            版本库的当前变化状态（新增了文件）
$ git add v1.md                            从工作区添加到缓存区
$ git status                            版本库的当前变化状态（文件已加入缓存区待提交到版本库）
$ git commit -m 'submit v1'                从缓存区添加到版本库

内容变更

$ echo 'version2 text content' >> v1.md        向v1.md文件追加内容
$ cat v1.md                                    查看追加的内容
$ git diff v1.md                            查看与缓存区之间的变更差异
$ git status                                版本库的当前当前变化状态（工作区文件被修改）
$ git add v1.md                                从工作区添加到缓存区
$ git commit -m 'submit v2'                    从缓存区添加到版本库

缓存区恢复及工作区恢复

$ echo 'version3 text content' >> v1.md        向v1.md文件追加内容
$ cat v1.md                                    查看追加的内容
$ git add v1.md                                从工作区添加到缓存区
$ git reset HEAD v1.md                        撤销对缓存区的修改
$ git checkout -- v1.md                        撤销对工作区的追加（没有--变成了切换分支）

切换版本

$ echo 'version3 text content' >> v1.md        向v1.md文件追加内容（增加测试数据）
$ git add v1.md                                从工作区添加到缓存区（增加测试数据）
$ git commit -m 'submit v3'                    从缓存区添加到版本库（增加测试数据）
$ git log                                    查看提交的记录
$ git log --pretty=oneline                    查看提交的简要记录
$ git reflog                                查看版本提交及回退记录
$ git reset --hard HEAD^                    v3回退一个版本v2（一个^代表回退一个版本）
$ git reset --hard *******                    v2回退到某个指定版本v3（******是commit id，git reflog或git log获取hash的前七位）
$ git reset --hard HEAD~2                    v3回退到前两个版本v1（上一操作是回退到最新的版本）

分支添加、切换、临时藏匿修改内容再建分支解决问题、合并、冲突解决

$ git checkout -b dev1    创建分支dev1并且切换到当前新创建的分支dev1下
$ git branch            查看当前工作区指向到分支的情况（指向到dev1）
$ git branch dev0        创建分支dev0
$ git checkout dev0        切换到dev0分支下
$ git branch            查看当前工作区指向到分支的情况（指向到dev0）
$ git checkout master    切换到主分支（指向到默认的master分支）
$ git branch -d dev0    删除分支dev0（当前分支指向的不是dev0才能删除）
$ git checkout dev1        切换到dev1分支下（开始dev1的开发任务）
$ echo 'version3.1 text content from branch dev1' >> v1.md        分支追加新的内容    
$ git stash             藏匿新追加的内容（如果不藏匿你要新增分支会把当前的变更内容也被指向）
$ git checkout -b dev2    创建分支dev2（突然要建个分支修bug，如果没有git stash是刚刚的追加会被指向过来且会在创建分支时候提醒你）
$ cat v1.md                查看追加的内容（此时发现刚刚在dev1追加的内容已经不见）
$ echo 'version3.1.1 text content from branch dev2' >> v1.md    分支追加新的内容
$ git add v1.md            从工作区添加到缓存区
$ git commit -m 'submit v3.1.1 bug dev2' 从缓存区添加到分支dev2
$ git checkout master    切换到主分支（指向到默认的master分支）
$ cat v1.md                查看追加的内容（虽然刚刚dev2提交了内容，但此时发现刚刚在dev2追加的内容已经不见）
$ git merge --no-ff -m "merge dev2" dev2        把分支dev2的修改合并到主分支master上（当前所在分支是master,--no-ff参数就可以用普通模式合并）
$ git branch -d dev2    删除分支dev2（bug解决完了就把它给删除然后回到dev1继续任务）
$ git checkout dev1        切换到dev1分支下
$ cat v1.md                查看内容（发现刚刚dev2的修改没有在这里看到）
$ git stash list        查看藏匿内容列表
$ git stash apply        恢复之前藏匿的内容
$ git stash drop        删除藏匿内容列表
$ git stash             藏匿变化的内容
$ git stash list        查看藏匿内容列表
$ git stash apply stash@{0}        恢复之前藏匿的内容且指定stash
$ git stash drop        删除藏匿内容列表
$ git stash             藏匿变化的内容
$ git stash pop            恢复藏匿的内容且删除藏匿内容列表
$ git add v1.md            从工作区添加到缓存区
$ git commit -m 'submit v3.1 after v3.1.1 bug' 从缓存区添加到dev1分支
$ git checkout master    切换到主分支（指向到默认的master分支）
$ git merge --no-ff -m "merge dev1" dev1        把分支dev1的修改合并到主分支master上（3.1.1和3.1的改动在此时出现冲突）
$ 打开文件把冲突的内容调整成你需要的
$ git add v1.md            从工作区添加到缓存区
$ git commit -m 'submit dev2 and dev1 modified conflict' 从缓存区添加到master分支
$ git log --graph --pretty=oneline --abbrev-commit          查看分支历史
